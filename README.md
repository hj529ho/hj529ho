# 스트릿 게임 개발자 정호진입니다. 👋

안녕하세요. Unity 클라이언트 개발자 정호진입니다.

저는 게임 개발에서 가장 중요한 가치로 협업과 지속 가능한 개발을 생각합니다. 
기능 하나를 빠르게 완성하는 것보다, 이후 업데이트에서 사이드 이펙트를 줄이고 팀 전체 생산성을 높일 수 있는 구조를 만드는 데 집중해왔습니다.

실무에서는 라이브 서비스 유지보수와 신작 개발을 모두 경험했습니다. 
라이브 프로젝트에서는 성능 저하와 운영 부담을 줄이는 작업을 맡아 Addressable 기반 리소스 로딩 구조를 적용했고, 메모리 사용량을 낮춰 안정성을 개선했습니다. 
신작에서는 초기 설계부터 글로벌 출시까지 참여하며 기능 확장과 운영을 고려한 구조로 개발을 진행했습니다.

저는 구현 단계에서 항상 "이 변경이 다음 기능 개발과 운영에 어떤 영향을 줄지"를 먼저 확인합니다. 
그래서 로직 분리, 명확한 책임 구조, 재사용 가능한 모듈 설계를 우선으로 두고 작업합니다. 
필요 시 클라이언트와 서버 경계를 재정의해 운영 리스크를 줄였고, 비즈니스 로직을 적절한 계층으로 이동시켜 패치 대응 속도를 높인 경험도 있습니다.

제 경력이 1년 단위로 짧게 보이는 구간이 있는 점은 사실입니다. 
다만 각 시기마다 프로젝트 단계(신규 개발, 라이브 운영, 출시 전환)에 맞춰 필요한 과제를 끝까지 책임지며, 구조 개선과 성능 안정화 같은 실질 성과를 남겨왔습니다. 
앞으로는 안정적으로 오래 근무하며 한 조직에서 더 긴 호흡으로 서비스의 중장기 로드맵까지 함께 책임지고, 누적된 맥락 위에서 더 큰 성과를 만드는 개발자가 되겠습니다.

저는 게임과 서브컬처를 깊이 좋아하는, 이른바 오타쿠 개발자입니다. 제게 게임 개발은 덕업일치이며, 업무 외 시간에도 팬게임을 포함한 취미 프로젝트를 꾸준히 만들며 실력을 다져왔습니다.
이 과정에서 단순히 게임을 완성하는 데서 그치지 않고, 팀이 더 효율적으로 일할 수 있는 방식까지 고민하게 됐습니다.
그 결과 GitHub의 RapkiSkillEditor_Unity에서는 기획자가 프로그래머 도움 없이 스킬 로직을 구성할 수 있는 노드 기반 에디터를 구현했고, 
반복 작업과 직군 간 의존성을 줄여 생산성 개선 가능성을 검증했습니다.
또한 hj529ho/ToonRenderResearch 레포지토리처럼 기술을 구현과 문서화로 축적하며, 개인적으로 쌓은 기술들을 실전에서도 쓸 수 있도록 꾸준히 정리하고 있습니다.

협업 측면에서는 개발자 관점만 고집하지 않고, 기획/아트 직군이 실제로 사용 가능한 형태를 함께 설계하는 데 강점이 있습니다. 
과거 팀 프로젝트에서 비개발 직군과의 협업을 통해 데이터 구조와 툴 사용성을 조정하며, 커뮤니케이션 비용을 줄이고 작업 흐름을 단순화한 경험이 있습니다.
또한 Jira와 애자일 프로세스를 실무에 적용해본 경험이 있어, 스크럼 기반 협업 환경에서도 일정 관리와 우선순위 조율에 능숙합니다.

팀에 합류하면 다음 세 가지로 기여하겠습니다.
첫째, 기능 단위 구현이 아니라 라이브 운영까지 고려한 구조 설계로 사이드 이펙트를 줄이겠습니다.
둘째, 반복 업무를 줄이는 툴과 공통 모듈을 구축해 팀 생산성을 높이겠습니다.
셋째, 협업 과정에서 직군 간 의존성을 낮춰 더 빠르고 안정적인 개발 사이클을 만들겠습니다.

저는 게임을 만드는 개발자이면서, 팀이 오래 안정적으로 개발할 수 있는 기반을 함께 만드는 개발자입니다. 
협업 중심의 구조 설계와 실무형 문제 해결력으로 완성도 높은 게임 개발에 기여하겠습니다.
